# 先生用ガイド — 双方向プログラミングスタジオ

中学校技術・家庭科(技術分野)「D 情報の技術」
**「ネットワークを利用した双方向性のあるコンテンツのプログラミング」** 向けの自作教材です。

- 生徒はブラウザだけで使えます(インストール不要・アカウント不要)
- ブロックプログラミングで「チャット」「自動応答ボット」「じゃんけん対戦」を作ります
- 発展課題として、ブロックから生成された**本物のJavaScript**の表示・改造もできます

---

## 1. 2つの動作モード

| モード | 条件 | できること |
|---|---|---|
| 🏝 **練習モード** | Firebase未設定のまま | 端末内だけで動作。「れんしゅうボット」相手に練習。**通信はしない** |
| 🌐 **ネットワークモード** | Firebase設定済み | クラス全員で本物のネットワーク通信(チャット・対戦) |

まず練習モードのまま授業の流れを確認し、本番前にFirebaseを設定するのがおすすめです。

---

## 2. Firebaseの設定(無料・所要約10分・一度だけ)

Google(Firebase)の無料データベースを「サーバー」として使います。
学級規模のチャットなら無料枠(同時接続100・保存1GB)で十分すぎるほどです。

### 手順

1. https://console.firebase.google.com を開き、Googleアカウントでログイン
   (学校のGoogle Workspaceアカウントでも個人アカウントでも可)
2. **「プロジェクトを作成」** → プロジェクト名は自由(例: `school-chat`)
   - Googleアナリティクスは **無効でOK**
3. 作成できたら、左メニュー **「構築」→「Realtime Database」→「データベースを作成」**
   - ロケーション: **アジア(asia-southeast1 など)** を選択
   - セキュリティルール: いったん「**ロックモード**」で開始
4. Realtime Database の **「ルール」タブ** を開き、中身をこのリポジトリの
   [`database.rules.json`](../database.rules.json) の内容に**まるごと置き換えて「公開」**
   (部屋のメッセージだけ読み書きでき、長すぎる投稿は拒否されるルールです)
5. ⚙(プロジェクトの設定)→「全般」→ 下の方の **「アプリを追加」→ Web(</>)** を選択
   - アプリのニックネームは自由。「Firebase Hosting」はチェック不要
6. 表示される `const firebaseConfig = { ... }` の中身を、
   このリポジトリの **`js/firebase-config.js`** に貼り付けて保存
   - GitHub上で直接編集する場合: リポジトリで `js/firebase-config.js` を開き ✏(Edit)→貼り付け→Commit
   - `databaseURL` が含まれていない場合は、Realtime Databaseの画面上部に表示される
     `https://～.firebasedatabase.app` のURLを `databaseURL: "..."` として自分で追記してください
7. 数分待ってページを再読み込みし、画面上部のバッジが
   **「🌐 ネットワークモード」** になっていれば完了です

### セキュリティについて

- `firebase-config.js` のAPIキーは「秘密の鍵」ではなく、Webアプリでは公開される前提のものです。
  データの保護はステップ4のルールで行います。
- チャット内容は同じ部屋コードを知っていれば誰でも読めます。
  **個人情報を書かない**指導とあわせてお使いください。
- 学期末などに: 先生用タブの「全消去」で部屋ごとに消すか、
  Firebaseコンソールでデータベースごと削除できます。

## 3. 授業での使い方(展開例: 2〜3時間)

### 準備
- 生徒にGitHub PagesのURLを配布(QRコードや学習支援アプリで)
- **部屋コード**を決めて指示する
  - 全体チャット体験: クラス全体で1部屋(例: `1A`)
  - ボット・対戦: **ペアやグループごとに別の部屋**(例: `1A01`〜`1A20`)
  - ※部屋を分けないと全員のボットが一斉に反応してにぎやかになりすぎます

### 展開例

| 時 | 内容 | 使うタブ |
|---|---|---|
| 1 | 双方向性のあるコンテンツの仕組み(サーバーの役割) → 全体チャット体験 → 情報モラル指導 | ①②(④で巡回) |
| 2 | ミッション1〜3: あいさつ送信(順次)→自動返信(イベント)→ことばに反応(分岐) | ③ |
| 3 | ミッション4〜5: おみくじボット(乱数)→じゃんけん対戦(総合) → 自由改良・相互評価 | ③ |

### 指導のポイント

- **🧪受信テスト**を使うと、相手がいなくても一人でボットの動作確認ができます(デバッグの習慣づけ)
- ボット同士が返事をし合うと「無限ループ」になり、自動停止して警告が出ます。
  **なぜ止まらなくなったのか**を考えさせると、情報システムの設計の学習になります
- **</>コードを見る**で、ブロックがJavaScriptに変換されていることを見せられます。
  得意な生徒には「改造モード」を発展課題に
- 作品は **💾保存** でファイル(.json)に書き出せます。次の時間に **📂読み込み** で続きから
  (同じ端末・同じブラウザなら自動保存でも復元されます)

### 先生用タブ(④)

- 部屋コードを入れると、その部屋のやりとりをリアルタイムでモニターできます(巡回指導・評価用)
- **📄ログを書き出し**: 記録をテキストファイルで保存(評価資料に)
- **🗑全消去**: その部屋のメッセージをデータベースから削除(授業の切り替え時に)

### 先生用タブのパスコード

④先生用タブは、開くときにパスコードの入力を求めます。

- **初期値: `sensei`** — 授業で使う前に必ず変更してください
- 変更方法: `js/firebase-config.js` の先頭近くにある
  `window.TEACHER_PASSCODE = "sensei";` の `"sensei"` を好きな文字列に書きかえるだけです
  (GitHubのWeb画面で ✏Edit→Commit でOK。数分でページに反映されます)
- 一度正解すると、そのブラウザのタブを開いている間は再入力不要です(タブやブラウザを閉じるとリセット)
- ⚠️ これは生徒のいたずらを防ぐための**簡易ガード**です。ページのソースコードを見れば
  分かってしまうため、本格的なセキュリティではありません。
  重要なデータの保護はFirebaseのルール(手順2-4)が担っています。

## 4. GitHub Pagesの更新方法

このリポジトリの `main` ブランチを更新すると、数分でページに反映されます。
`js/firebase-config.js` の編集もGitHubのWeb画面から可能です(手順2-6参照)。

## 5. トラブルシューティング

| 症状 | 対処 |
|---|---|
| バッジが「練習モード」のまま | `js/firebase-config.js` の貼り付け内容を確認(特に `databaseURL`)。ブラウザをスーパーリロード(Ctrl+Shift+R) |
| 「せつぞく」してもメッセージが届かない | 部屋コードが完全に一致しているか確認(全角/半角に注意) |
| プログラムが勝手に止まる | 短時間に送信しすぎ(無限ループ検知)。⚡受け取ったとき→✉送信 のループになっていないか確認 |
| 学校のネットワークでつながらない | `*.firebasedatabase.app` `*.googleapis.com` `www.gstatic.com` への接続許可を管理者に依頼 |
